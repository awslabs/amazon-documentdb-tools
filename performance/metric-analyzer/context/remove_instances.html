Excessive Instance Deployment Considerations

Amazon DocumentDB is designed for 99.99% availability when deployed across two or more AZâ€™s. Replica instances work well for read scaling because they are fully dedicated to read operations on your cluster volume. However running an Amazon DocumentDB cluster with more than three instances (one primary and two replicas) does not increase the availability beyond 99.99% and can increase costs.

Considerations before removing instances:

  1. Read scaling needs: Determine if your current read workload actually requires the additional replica instances.
  2. Regional distribution: Verify that the current instances are distributed across multiple Availability Zones for maximum resilience.
  3. Maintenance strategy: Consider how instance removal might affect your maintenance procedures.
  4. Future growth plans: Assess if anticipated workload growth justifies keeping additional instances.

Additional considerations:

  1. For production workloads, it is recommended to run a cluster with three instances (one primary, two replicas), but no less than two (one primary, one replica)
  2. Ensure instances are distributed across different Availability Zones
  3. Configure read preference to secondaryPreferred to maximize read scaling
  4. Consider increasing instance size instead of adding more instances if additional capacity is needed
