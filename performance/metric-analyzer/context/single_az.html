Single Instance Deployment Considerations

Running a DocumentDB cluster with only a single instance provides no high availability and no read scaling capabilities. For production workloads, it is recommended to deploy a cluster with at least one replica instance.

Considerations before adding replica instances:

  1. High availability needs: Single-instance clusters have no automatic failover capability, resulting in longer downtime during instance failures.
  2. Read scaling requirements: Without replicas, all read operations must be processed by the primary instance.
  3. Maintenance impact: With replicas, maintenance operations can be performed with minimal downtime.

Additional considerations:

  1. For production workloads, deploy at least one replica instance (two total instances)
  2. For critical workloads, consider deploying two replica instances (three total instances)
  3. Amazon DocumentDB clusters can be stopped and started, helping to manage costs for development and test environments
  4. Configure read preference to secondaryPreferred to utilize replica instances for read operations
  5. Monitor replica lag to ensure acceptable data consistency
